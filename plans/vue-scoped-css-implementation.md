# Vue Scoped CSS Implementation Plan

## Overview

This plan describes how to implement `<style scoped>` support in vue-lynx by bridging Vue's scope ID system to Lynx's native `cssId` scoping. The key insight is to **reuse the scope ID generated by `rspack-vue-loader`** rather than implementing our own hash algorithm.

## Core Strategy

1. **Extract scope ID from vue-loader**: Capture the `id` query parameter from `?vue&type=script&id=xxx&scoped=true` requests
2. **Convert to Lynx cssId**: Transform Vue's `data-v-xxx` format to Lynx's numeric `cssId`
3. **Transform CSS selectors**: Convert `[data-v-xxx]` attribute selectors to class-based selectors
4. **Bridge to runtime**: Pass scope information from BG thread to MT via ops

### 1.2 Create vue-scope-bridge.ts

**New File**: `packages/vue-lynx/plugin/src/loaders/vue-scope-bridge.ts`

```typescript
import type { Rspack } from '@rsbuild/core';
import * as qs from 'querystring';

export default function vueScopeBridgeLoader(
  this: Rspack.LoaderContext,
  source: string,
): string {
  this.cacheable(true);
  
  if (!this.resourceQuery?.includes('vue') || !this.resourceQuery?.includes('type=script')) {
    return source;
  }
  
  const query = qs.parse(this.resourceQuery.slice(1));
  const scopeId = query.id as string | undefined;
  const hasScoped = query.scoped === 'true';
  
  if (scopeId && hasScoped) {
    const cssId = parseInt(scopeId, 16);
    
    // Store mapping on compiler for later use
    const compiler = this._compiler as any;
    if (!compiler.__vueLynxScopeMap) {
      compiler.__vueLynxScopeMap = new Map();
    }
    compiler.__vueLynxScopeMap.set(this.resourcePath, {
      scopeId: `data-v-${scopeId}`,
      cssId,
      hasScoped,
    });
    
    const marker = `/*__VUE_LYNX_SCOPE__${scopeId}__*/`;
    return marker + source;
  }
  
  return source;
}
```

## Phase 2: CSS Selector Transformation

### 2.1 Create PostCSS Plugin

**New File**: `packages/vue-lynx/plugin/src/postcss-plugins/vue-scope-selector-plugin.ts`

```typescript
import type { AcceptedPlugin } from 'postcss';
import selectorParser from 'postcss-selector-parser';

export function vueScopeSelectorPlugin(): AcceptedPlugin {
  return {
    postcssPlugin: 'vue-scope-selector',
    Once(root) {
      const from = root.source?.input.file;
      if (!from?.includes('.vue')) return;
      
      root.walkRules((rule) => {
        if (rule.parent?.type === 'atrule' && rule.parent.name === 'keyframes') {
          return;
        }
        
        const selectorRoot = selectorParser().astSync(rule.selector);
        
        selectorRoot.each((selector) => {
          selector.walk((node) => {
            if (node.type === 'attribute' && node.attribute.startsWith('data-v-')) {
              const scopeClass = node.attribute.replace('data-v-', 'v-');
              const classNode = selectorParser.className({ value: scopeClass });
              node.replaceWith(classNode);
            }
          });
        });
        
        rule.selector = selectorRoot.toString();
      });
    },
  };
}
```

## Phase 3: Runtime Bridge

### 3.1 Create scope-bridge.ts

```typescript
import { getCurrentInstance } from '@vue/runtime-core';

export function getCurrentScopeId(): string | undefined {
  const instance = getCurrentInstance();
  if (!instance) return undefined;
  return (instance.type as any).__scopeId;
}

export function scopeIdToCssId(scopeId: string): number {
  const hex = scopeId.replace(/^data-v-/, '');
  return parseInt(hex, 16);
}
```

## Key Design Decisions

1. **Reuse vue-loader's scope ID**: Avoids reimplementing hash algorithm, maintains compatibility with Vue ecosystem

2. **PostCSS-based CSS transform**: More reliable than string replacement, handles complex selectors correctly

3. **Two-phase scoping**: Compile-time (selector transform) + runtime (cssId assignment)

4. **Minimal runtime overhead**: Scope ID only processed for elements in scoped components

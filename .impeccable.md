## Design Context

### Users

Vue and Lynx developers use the website while learning APIs, evaluating compatibility, and running examples in Web or native environments. The interface should help them understand behavior and make technical comparisons without distracting from the documentation.

### Brand Personality

Precise, pragmatic, and forward-looking. Vue Lynx is experimental, but its documentation should feel trustworthy and engineered rather than flashy or provisional.

### Aesthetic Direction

Stay visually native to the existing Rspress documentation theme. Use Vue green sparingly for active states and capability signals, preserve the current typography and compact spatial rhythm, and support both light and dark themes. Avoid oversized controls, floating global UI, decorative gradients, and generic dashboard styling.

### Design Principles

1. Make interaction scope visible: page-wide actions must look page-wide; entry-specific state must look local.
2. Keep documentation primary: controls should be compact, calm, and subordinate to headings and examples.
3. Explain fallbacks in place: unsupported features remain usable in VDOM and state why without interrupting the reader.
4. Use Vue green as a functional accent, not decoration, with accessible contrast and non-color status cues.
5. Adapt components to their container and input method, including 44px touch targets and visible keyboard focus.

# HackerNews Slopfork

我们 Fork 了 Evan You 的经典 [Vue HackerNews 2.0](https://github.com/vuejs/vue-hackernews-2.0) —— 那个 Vue 最经典的「真实世界」示例 —— 作为 Vue Lynx 的能力验证基准。由于 Vue Lynx 基于 Vue 3，我们使用了社区的 [Vue HackerNews 3.0](https://github.com/raukaute/vue-hackernews-3.0) 移植版作为参考。

**复用信心：高。** 大部分代码可以直接迁移 —— 组件逻辑、Vue Router、Composition API，甚至 `<Transition>`/`<TransitionGroup>` 都完全一致。唯一需要修改的是平台相关部分：HTML 元素 → Lynx 元素、`@click` → `@tap`、`createWebHistory()` → `createMemoryHistory()`。

### Tailwind CSS 版本

在第一轮迭代中，我们让 AI 用 Tailwind CSS (`@lynx-js/tailwind-preset`) 重写了样式。同时集成了 TanStack Vue Query 作为 Vuex 4 的现代替代方案。

## 改动概览

Vue HackerNews 3.0 参考实现是一个基于 Webpack 的 SSR 应用，使用 Vue 3 + Vuex 4 + axios。移植到 Lynx 时的改动如下：

| 方面 | Vue 3 参考 | Vue Lynx |
|------|-----------|----------|
| **模板元素** | `<div>`、`<span>`、`<img>`、`<a>` | `<view>`、`<text>`、`<image>`、tap 事件 |
| **事件处理** | `@click` | `@tap` |
| **路由历史** | `createWebHistory()` | `createMemoryHistory()` |
| **滚动** | 浏览器原生滚动 | `<scroll-view>` 组件 |
| **状态管理** | Vuex 4 | TanStack Query (Tailwind) / 相同模式 (CSS) |
| **HTTP 客户端** | axios | `globalThis.fetch` |
| **构建工具** | Webpack 5 + Express | Rspeedy |
| **SSR** | 完整 SSR + 注水 | 无（Lynx 不使用 SSR）|

### 值得注意的适配：

- **没有 `v-html`**：Lynx 没有 HTML 解析器，所以我们用 `stripHtml()` 工具函数将 HTML 评论转换为纯文本
- **`<router-link>`**：用 Vue Router 的 `custom` slot API 包装，因为 Lynx 没有 `<a>` 标签
- **SVG spinner**：替换为 CSS `border` + `animation` spinner（Lynx 不支持 SVG）

## 完全一致的部分

- **Vue Composition API**：`ref()`、`computed()`、`watch()`、`<script setup>` —— 完全一致
- **Vue Router API**：`useRoute()`、`useRouter()`、`router.push()`、`<RouterView>` —— 完全一致
- **`<Transition>` / `<TransitionGroup>`**：CSS transition 类名完全一致
- **递归组件**：`<Comment>` 自引用渲染嵌套评论 —— 完全一致
- **模板指令**：`v-if`/`v-for`/`v-bind`/`{{ }}` —— 全部一致
- **CSS**：SCSS、`@keyframes`、`:class` 绑定、media queries —— 全部可用

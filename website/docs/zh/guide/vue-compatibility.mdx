# Vue 特性兼容性

:::tip
本页逐一介绍了 Vue 核心特性在 Lynx 上的支持情况。如需了解 Vue Lynx 双线程架构背后的设计理念，请阅读[理解双线程模型](/zh/guide/introduction#understanding-the-dual-thread-model)。
:::

由于 Vue Lynx 基于官方 Vue 3 运行时核心构建，它的目标是开箱即用地支持完整的 Vue 3 API。核心渲染路径——Composition API、SFC、响应式、模板指令——与标准 Vue 的行为完全一致。

不过，Vue 的 API 面很广，我们尚未来得及全面验证每一个细节。以下列出的是我们已确认可以在 Lynx 上顺畅运行的关键 Vue 特性——绝大多数无需对标准 Vue 代码库做任何改动。如果存在 Lynx 特有的注意事项，会在对应位置标注说明。

## SFC CSS 特性

普通 `<style>` 块、导入的 `.css` 文件，以及 [`<style module>`](https://vuejs.org/api/sfc-css-features.html#css-modules) 均可在 Lynx 上正常使用。`<style scoped>` 和 CSS 中的 `v-bind()` 尚未支持——如果你需要这些特性，请在 [#78](https://github.com/Huxpro/vue-lynx/issues/78) 和 [#79](https://github.com/Huxpro/vue-lynx/issues/79) 中投票。

:::details 特性支持详情

| 特性 | 状态 |
|---------|--------|
| `<style>`（普通） | 可用 |
| 导入的 `.css` 文件 | 可用 |
| `<style module>` | 可用 |
| `<style scoped>` | 即将支持（[#78](https://github.com/Huxpro/vue-lynx/issues/78)） |
| `<style>` 中的 `v-bind()` | 即将支持（[#79](https://github.com/Huxpro/vue-lynx/issues/79)） |

对于动态样式，可以使用计算属性 `:style` 绑定作为 `v-bind()` 的替代方案：
```vue
<script setup>
const color = ref('#1565c0')
const style = computed(() => ({ color: color.value }))
</script>
<template>
  <text :style="style">Dynamic color</text>
</template>
```
:::

## 组件 v-model

Vue 的 `v-model` 可以在父子组件之间创建双向绑定。子组件使用 [`defineModel()`](https://vuejs.org/api/sfc-script-setup.html#definemodel)（Vue 3.4+）声明 model prop，父组件通过 `v-model` 进行绑定。

该示例演示了：
1. **默认 model** —— 使用 `defineModel<number>()` 配合 `v-model="count"` 实现计数器
2. **具名 model** —— 使用 `defineModel('title')` + `defineModel('body')` 配合 `v-model:title` / `v-model:body`
3. **手动输入绑定** —— 原生 `<input>` 的替代方案（参见下方注意事项）

:::warning 注意
原生 `<input>` 元素上的 `v-model` 尚未支持。Lynx 的输入控件使用同步的主线程 API（`getValue()` / `setValue()`），这些 API 在 Vue 运行所在的后台线程中无法访问。请改用 `:value` + `@input` 手动绑定：
```vue
<input :value="text" @input="(e) => text = e.detail.value" />
```
:::

## Transition

Vue 的 `<Transition>` 和 `<TransitionGroup>` 组件在元素插入或移除时应用进入/离开动画。

:::warning
`<Transition>` 和 `<TransitionGroup>` 目前处于**实验性**阶段。请始终传入明确的 `:duration` prop——因为后台线程中无法使用 `getComputedStyle()`。`<TransitionGroup>` 中的移动（FLIP）动画不受支持，因为 `getBoundingClientRect()` 不可用。

## Options API

Vue 3 在 Composition API 之外同时提供了 Options API 以保持向后兼容。
默认情况下，Vue Lynx 启用了 Options API（插件中 `optionsApi: true`），但你可以将其禁用以减小打包体积：

```ts title="lynx.config.ts"
pluginVueLynx({
  optionsApi: false, // tree-shake Options API 运行时（约 9 kB）
})
```

## 尚未支持的特性

部分 Vue 内置特性尚未适配双线程原生环境：

| 特性 | 原因 | 替代方案 |
|---------|--------|-------------|
| `<KeepAlive>` | 需要一个内部存储容器（`createElement('div')`），在原生环境中没有对应实现 | 手动缓存状态 |
| `<Teleport>` | 需要 `querySelector` 来解析字符串目标，在双线程架构中不可用 | 在目标位置使用条件渲染 |
| `<Transition>` 自动时长 | 后台线程中无法使用 `getComputedStyle()` | 始终传入明确的 `:duration` prop |

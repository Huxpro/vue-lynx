# 首屏直出（IFR）

首屏直出（Instant First-Frame Rendering，IFR）让页面在加载完成的那一刻就显示真实内容——没有白屏，也没有中间加载态。它是 Lynx 官方 React 框架自带的同名优化在 Vue Lynx 上的移植。平台层面的背景介绍见 [Lynx IFR 指南](https://lynxjs.org/guide/interaction/ifr)。

## 白屏从哪来

在 Vue Lynx 默认的双线程管线中，所有可见内容都由后台线程产出：

```
  没有 IFR 时：

  ┌───────────────┐   ┌─────────────────────┐   ┌───────────────┐
  │    主线程     │   │      后台线程       │   │    主线程     │
  │  （空页面）   │──▶│（Vue 渲染并发送     │──▶│（应用 ops，   │
  │               │   │  ops 批次）         │   │  完成首帧）   │
  └───────────────┘   └─────────────────────┘   └───────────────┘
        ▲                                               │
        └────────── 用户盯着一帧白屏 ───────────────────┘
```

主线程在 `loadTemplate` 期间只创建了一个空页面，然后开始等待：后台线程要启动、执行应用 bundle、跑完 Vue 的首次渲染，再把产生的 ops 跨线程发回来，屏幕上才会出现内容。

## Vue Lynx 中的 IFR 工作原理

启用 IFR 后，主线程 bundle 会携带完整的 Vue 运行时和你的应用代码。首屏在 `loadTemplate` 期间、**在主线程上**同步渲染完成：

```
  启用 IFR 后：

  ┌────────────────────────────────────────────────┐
  │                    主线程                      │
  │  loadTemplate ──▶ Vue 渲染 ──▶ 直接绘制首帧    │
  └────────────────────────────────────────────────┘
                          │（并行进行）
  ┌────────────────────────────────────────────────┐
  │                   后台线程                     │
  │  启动 ──▶ 渲染同一应用 ──▶ hydration           │
  └────────────────────────────────────────────────┘
```

1. **主线程直出首帧。** Lynx 调用 `renderPage` 时，Vue 应用直接在主线程上挂载，渲染产生的元素操作通过 Element PAPI 立即执行——首帧在任何后台 JavaScript 运行之前就已上屏。每个已应用的 ops 批次同时被记录下来。
2. **后台渲染。** 后台线程启动后用同样的数据渲染同一个应用。由于两个线程执行完全相同的确定性代码，它产生的元素 id 和事件 sign 与主线程渲染逐一对齐。
3. **Hydration。** 后台线程的初始 ops 批次会与记录的 ops 流做比对，而不是再次应用：
   - 完全一致的批次会被**跳过**——内容已经在屏幕上；
   - 值级差异（文本、样式、属性）会被**原地打补丁**；
   - 结构性差异则拆除首屏树、按后台 ops 重建——正确性从不依赖两次渲染一致，只有性能收益依赖。

Hydration 完成后，树的所有权交给后台线程，后续更新都走正常的 ops 管线。首帧期间绑定的事件使用与后台线程一致的确定性 sign，点击无需任何重新绑定即可路由到你的 Vue 事件处理函数。

## 启用方式

在插件选项中开启：

```ts title="lynx.config.ts"
import { defineConfig } from '@lynx-js/rspeedy'
import { pluginVueLynx } from 'vue-lynx/plugin'

export default defineConfig({
  plugins: [
    pluginVueLynx({
      enableIFR: true,
    }),
  ],
})
```

典型的组件树无需修改任何应用代码。

## 编写 IFR 友好的首屏

首屏渲染会执行两次——每个线程一次——因此它必须表现得像输入的纯函数：

- **保持渲染输出确定性。** 首屏渲染路径上避免 `Math.random()`、`Date.now()` 或依赖线程的分支。如果两次渲染出现结构性分歧，Vue Lynx 会检测到并回退为完整的后台渲染——页面依然正确，但该屏失去 IFR 收益（dev 构建会输出警告）。
- **副作用放进生命周期钩子。** `onMounted`、`onUnmounted` 等钩子在主线程渲染期间不会执行（与 ReactLynx 中 `useEffect` 不在主线程触发一致）。数据请求、定时器、订阅只会在后台线程运行一次。
- **首屏只画同步数据。** IFR 绘制的是初始数据能渲染出的内容。依赖网络请求的内容仍要等后台线程取回后出现——可以为这些区域准备骨架屏，让 IFR 消除它们周围的白屏。
- **后台线程就绪前的事件会被丢弃。** 从首帧绘制到后台线程完成启动之间有一个很短的窗口（通常几毫秒），常规事件处理函数在那之后才生效。[主线程脚本](/zh/guide/main-thread-script)事件不受影响——它们本来就在主线程上。

## 取舍

- 主线程 bundle 大约增加一个 Vue 运行时加应用代码的体积，模板变大、`loadTemplate` 的解析时间变长。对内容型首屏来说，感知延迟的收益通常占优；建议按自己的应用实测。
- 当 bundle 本身需要异步加载，或整个首屏都依赖异步数据时，IFR 无能为力——与 ReactLynx IFR 的约束相同。

## 试一试

`hello-world` 示例已默认启用 IFR：

```bash
git clone https://github.com/Huxpro/vue-lynx
cd vue-lynx && pnpm install && pnpm build
cd examples/hello-world && pnpm build
```

检查 `dist/main.lynx.bundle`，`lepusCode` 段现在包含了你的应用——在 LynxExplorer 中加载它，完整首屏会在 `background.js` 执行任何一行代码之前绘制出来。

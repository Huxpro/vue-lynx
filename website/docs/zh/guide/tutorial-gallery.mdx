# 教程：产品画廊

在本教程中，我们将使用 Vue 3 和 Lynx 一起构建一个产品画廊页面。本教程不假设你有任何 Lynx 相关的知识。你将在本教程中学到的技术是构建任何 Lynx 页面和应用的基础。

:::note

本教程专为喜欢边做边学、希望快速做出实际成果的人设计。如果你更喜欢逐步学习每个概念，请从[描述 UI](https://lynxjs.org/next/guide/ui/elements-components) 开始。

:::

## 我们要构建什么？

一个具有精美瀑布流布局的家具画廊，支持点击收藏交互、自动滚动和自定义滚动条——由主线程脚本驱动，实现丝滑流畅的性能体验。每个章节都在前一个章节的基础上逐步构建。

## 你的第一个组件：图片卡片

现在，让我们从创建第一个图片卡片开始，它将是本页面的核心部分。

<Go example="gallery" defaultFile="src/ImageCard/ImageCard.vue" entry="src/ImageCard" defaultEntryName="ImageCard" />

很好，你现在可以看到图片卡片已经显示出来了。这里我们使用 [`<image>`](https://lynxjs.org/next/api/elements/built-in/image) 元素来显示图片。你只需要给它一个宽度和高度（或者像这里一样指定 `aspectRatio` 属性），它就会自动调整大小以适应指定的尺寸。该组件可以通过 `defineProps` 接收一个 `picture` 属性，允许你更改它显示的图片。

:::details 图片的 src 属性
Lynx 的 `<image>` 元素可以接受本地相对路径作为 `src` 属性来渲染图片。本页面中的所有图片都来自本地，这些路径在使用前需要先导入。但如果你的图片存储在网上，你可以轻松地将它们替换为网络图片地址。
:::

:::info Vue Lynx 入口点
与标准 Vue 类似，Vue Lynx 使用 `createApp(Component).mount()`。不同之处在于没有 DOM 元素可供挂载——`mount()` 无需参数即可调用，它会自动附加到 Lynx 页面根节点。
:::

## 添加交互：收藏图片卡片

我们可以在右上角添加一个白色小爱心，将其作为图片卡片的收藏按钮。这里我们实现了一个名为 `LikeIcon` 的小组件：

<Go example="gallery" defaultFile="src/Components/LikeIcon.vue" entry={["src/Components", "src/LikeCard"]} defaultEntryName="LikeCard" />

我们希望每张卡片都能记住是否被收藏过，因此我们添加了 `isLiked` 作为其内部数据。它可以使用这个内部数据来保存你的更改。

```vue title="LikeIcon.vue" {2}
<script setup lang="ts">
const isLiked = ref(false);
</script>
```

`ref(false)` 创建一个响应式变量。当 `isLiked.value` 发生变化时，Vue 的响应式系统会自动重新渲染该组件。

然后我们在外层 `<view>` 上添加 `@tap` 事件，这样当用户点击爱心时，就会触发该事件并改变 `isLiked` 的状态：

```vue title="LikeIcon.vue" {2,5}
<script setup lang="ts">
function onTap() {
  isLiked.value = true;
}
</script>

<template>
  <view class="like-icon" @tap="onTap">
    ...
  </view>
</template>
```

:::details 什么是 "@tap"？
Vue Lynx 使用与 Web 端 Vue 相同的 `@eventName` 语法，但 Lynx 有自己的一套事件——用 `@tap` 代替 `@click`，用 `@scroll` 代替原生滚动事件等。在[事件处理](https://lynxjs.org/next/guide/interaction/event-handling)页面了解更多信息。
:::

最后，我们使用 `isLiked` 来控制收藏效果。因为 `isLiked` 是响应式的，`LikeIcon` 会响应它的变化，变成红色爱心图标。用于渲染涟漪动画的 `<view>` 元素通过 `v-if` 进行条件渲染：

```vue title="LikeIcon.vue"
<template>
  <view class="like-icon" @tap="onTap">
    <view v-if="isLiked" class="circle" />
    <view v-if="isLiked" class="circle circleAfter" />
    <image :src="isLiked ? redHeart : whiteHeart" class="heart-love" />
  </view>
</template>
```

接着我们将 `LikeIcon` 和 `ImageCard` 组合成 `LikeImageCard` 组件（请参阅上方查看器中的代码）。

为了给收藏功能提供更好的视觉交互效果，我们在 `gallery.css` 中添加了 CSS 动画。你也可以在[动画](https://lynxjs.org/next/guide/styling/animation)章节了解更多关于动画的内容，并替换为你喜欢的样式！

## 通过元素方法实现自动滚动

如果你想创建一个桌面照片墙，就需要为这个页面添加自动滚动功能。你的图片将被缓慢地自动滚动，让你轻松浏览更多图片：

<Go example="gallery" defaultFile="src/GalleryAutoScroll/Gallery.vue" entry="src/GalleryAutoScroll" defaultEntryName="GalleryAutoScroll" />

```vue title="GalleryAutoScroll/Gallery.vue" {3-12}
<script setup lang="ts">
import { onMounted, nextTick } from 'vue-lynx';

onMounted(() => {
  nextTick(() => {
    lynx
      .createSelectorQuery()
      .select('[custom-list-name="list-container"]')
      .invoke({
        method: 'autoScroll',
        params: { rate: '60', start: 'true' },
      })
      .exec();
  });
});
</script>
```

我们使用 `onMounted` 结合 `nextTick` 在列表渲染完成后调用 [`autoScroll`](https://lynxjs.org/next/api/elements/built-in/list.html#autoscroll) 方法。我们使用 `lynx.createSelectorQuery()` 通过 `custom-list-name` 属性选择列表元素并调用原生方法。

:::details 什么是 "invoke"？
在 Lynx 中，所有原生元素都有一组可以调用的"方法"。与 Web 不同的是，这种调用是异步的，类似于消息传递。你需要使用 `invoke` 并传入方法名和参数来调用它们。
:::

:::details onMounted + nextTick
`onMounted` 在组件挂载时运行一次。我们在其中调用 `nextTick` 来等待主线程完成原生元素的创建。在 Vue Lynx 中，`nextTick` 经过增强，它会等待跨线程操作刷新完成——而不仅仅是 Vue 的内部调度器——因此当回调触发时，元素保证已经存在。
:::

## 来一个自定义滚动条怎么样？

和大多数应用一样，我们可以为这个页面添加一个滚动条来指示还有多少图片未显示。但我们可以做得更好！例如，我们可以用自己喜欢的样式替换 `<list>` 的默认进度条：

<Go example="gallery" defaultFile="src/GalleryScrollbar/NiceScrollbar.vue" entry="src/GalleryScrollbar" defaultEntryName="GalleryScrollbar" />

`NiceScrollbar` 组件通过 `defineExpose` 暴露了一个 `adjustScrollbar` 方法，这允许父组件调用子组件的方法。与之前用于添加收藏功能的 `@tap` 事件类似，我们在 `<list>` 上添加了 [`@scroll`](https://lynxjs.org/next/api/elements/built-in/list.html#scroll) 事件，该事件会在 `<list>` 元素滚动时触发。为了让滚动条对滚动事件的响应更快，我们将 [`scroll-event-throttle`](https://lynxjs.org/next/api/elements/built-in/list.html#scroll-event-throttle) 设置为 0。

:::details list-item 的 estimated-main-axis-size-px
你可能注意到了 `<list-item>` 上的这个属性。它用于估算元素在 `<list>` 中尚未渲染时在主轴上的大小。当我们添加滚动条时这非常有用，因为我们需要知道滚动条应该有多长才能覆盖所有元素。

我们提供了一个工具方法，可以根据当前布局和图片尺寸来估算大小：

```ts title="utils.ts"
export const calculateEstimatedSize = (
  pictureWidth: number,
  pictureHeight: number,
): number => {
  const galleryPadding = 20;
  const galleryMainAxisGap = 10;
  const gallerySpanCount = 2;
  const galleryWidth = SystemInfo.pixelWidth / SystemInfo.pixelRatio;
  const itemWidth = (galleryWidth - galleryPadding * 2 - galleryMainAxisGap)
    / gallerySpanCount;
  return (itemWidth / pictureWidth) * pictureHeight;
};
```
:::

到目前为止，我们已经有了一个完整的页面！但你可能注意到我们添加的滚动条在滚动过程中仍有些延迟，响应不够灵敏。这是因为我们的调整仍然发生在后台线程上，而不是响应触摸滚动的主线程上。

:::details 什么是后台线程和主线程？
Lynx 最大的特点是其双线程架构。你可以在 [JavaScript 运行时](https://lynxjs.org/next/guide/scripting-runtime/index.html#javascript)中找到更详细的介绍。
:::

## 更灵敏的滚动条

为了优化滚动条的性能，我们需要引入[主线程脚本 (MTS)](https://lynxjs.org/next/react/main-thread-script.html) 来[在主线程上处理事件](https://lynxjs.org/next/guide/interaction/event-handling.html#main-thread-event-processing)，将我们在上一步中对滚动条高度和位置的调整从后台线程迁移到主线程。

为了让你更清楚地看到对比效果，我们保留了两个滚动条：

<Go example="gallery" defaultFile="src/GalleryScrollbarCompare/Gallery.vue" entry="src/GalleryScrollbarCompare" defaultEntryName="GalleryScrollbarCompare" />

MTS 滚动条使用 `:main-thread-ref` 将一个 `MainThreadRef` 绑定到元素上，允许主线程直接操作它，无需任何后台线程的往返通信。

现在你应该可以看到，左侧的滚动条（darkkhaki 色）使用主线程脚本控制，与我们之前实现的右侧滚动条相比更加流畅和灵敏。如果你在其他 UI 中遇到需要立即更新的场景，可以尝试这种方法。

Vue 中 MTS 的关键概念：

- `useMainThreadRef(null)` — 创建一个可以在主线程上解析的 ref
- `:main-thread-bindscroll="ctx"` — 将 worklet 处理函数绑定到滚动事件（在主线程上运行，零线程切换）
- `.toJSON()` — 将 ref 序列化以进行跨线程传输（包含内部 `_wvid` 标识符）

## React 与 Vue 快速对照表

| 概念             | React Lynx                           | Vue Lynx                           |
| ---------------- | ------------------------------------ | ---------------------------------- |
| 创建应用         | `root.render(<App />)`               | `createApp(App).mount()`           |
| 状态             | `const [x, setX] = useState(val)`    | `const x = ref(val)`               |
| 更新状态         | `setX(newVal)`                       | `x.value = newVal`                 |
| 模板 ref         | `const ref = useRef(null)`           | `const el = ref(null)`             |
| 生命周期         | `useEffect(() => ..., [])`           | `onMounted(() => ...)`             |
| 类名             | `className="foo"`                    | `class="foo"`                      |
| 条件渲染         | `{cond && <el />}`                   | `<el v-if="cond" />`               |
| 列表渲染         | `{arr.map(x => <el key={x} />)}`     | `<el v-for="x in arr" :key="x" />` |
| 点击事件         | `bindtap={handler}`                  | `@tap="handler"`                   |
| 滚动事件         | `bindscroll={handler}`               | `@scroll="handler"`                |
| MTS 滚动         | `main-thread:bindscroll={fn}`        | `:main-thread-bindscroll="ctx"`    |
| MTS ref          | `main-thread:ref={ref}`              | `:main-thread-ref="ref"`           |
| 主线程 ref Hook  | `useMainThreadRef(null)`             | `useMainThreadRef(null)`           |
| 暴露给父组件     | `forwardRef` + `useImperativeHandle` | `defineExpose({ method })`         |
| Props            | `function Comp(props: { x: T })`     | `defineProps<{ x: T }>()`          |

# 什么是 Vue Lynx？

**Vue Lynx** 是一个面向 [Lynx](https://lynxjs.org) 的 Vue 3 自定义渲染器，让你可以使用熟悉的 Vue Composition API、单文件组件和响应式数据模型来构建原生 Lynx 应用。

如果你已经了解 Vue 3 并且熟悉 Lynx，那么你已经知道如何使用 Vue Lynx 了。

## 主要特性

- **熟悉的 Vue 3 API** — 使用 `ref()`、`computed()`、`v-for`、`v-if`、SFC `<script setup>` 以及你喜爱的 Vue 一切特性。
- **跨平台原生渲染** — Lynx 在移动端和桌面端提供高性能原生渲染引擎，同时也提供一流的 [Web 渲染](https://lynxjs.org/next/guide/web-rendering.html)支持。你的 UI 永远不是 WebView。
- **双线程架构** — Vue 运行在后台线程；主线程负责原生渲染。这种分离使你的应用保持流畅响应。对于延迟敏感的交互（拖拽、滚动），[主线程脚本](/zh/guide/main-thread-script) 允许你用 `'main thread'` 标记函数，使其直接在主线程上运行，实现零跨线程延迟。

## 面向 Vue Web 开发者

如果你是从 Web 端的 Vue 转过来的，你所掌握的大部分知识都可以直接复用。以下是主要的差异。

### 更改你的 `import`

Vue Lynx 重新导出了 Vue API，因此你可以将 `vue` 替换为 `vue-lynx`：

```diff
- import { ref, computed, onMounted } from 'vue';
+ import { ref, computed, onMounted } from 'vue-lynx';
```

### 不同的元素

Lynx 提供原生元素而非 HTML。使用 `<view>`、`<text>`、`<image>` 等来替代 `<div>`、`<span>`、`<img>`：

```diff
- <div class="card">
-   <span>{{ message }}</span>
- </div>
+ <view class="card">
+   <text>{{ message }}</text>
+ </view>
```

:::info
文本内容必须包裹在 `<text>` 中 — Lynx 不支持在 `<view>` 内直接放置裸文本节点。
:::

### 不同的事件

Lynx 使用自己的事件系统。使用 `:bindeventname` 表示可冒泡的事件，使用 `:catcheventname` 表示不冒泡的事件：

```diff
- <div @click="handleClick" @touchstart="onTouch">
+ <view :bindtap="handleClick" :bindtouchstart="onTouch">
```

更多详情请参阅 [Lynx 事件处理指南](https://lynxjs.org/guide/interaction/event-handling.html)。

### 没有 `document` 和 `window`

Lynx 不提供浏览器全局对象。依赖 `document` 或 `window` 的库需要经过适配才能使用。Lynx 通过 [`lynx`](https://lynxjs.org/next/api/lynx-api/lynx.html) 全局对象提供自己的 [API](https://lynxjs.org/next/api/lynx-api/lynx-api-overview.html)：

- [全局 API](https://lynxjs.org/next/api/lynx-api/global.html) 如 `setTimeout()` 和 `console.log()` 在两个线程上都可用，与 Web 端一致。
- 类 DOM 查询（如 `querySelector`、`getBoundingClientRect`）可通过[异步后台线程 API](https://lynxjs.org/next/api/lynx-api/nodes-ref.html) 或[同步主线程 API](https://lynxjs.org/next/api/lynx-api/main-thread.html) 使用，具体取决于你需要在哪个线程上执行。
- 平台原生能力可通过 [NativeModules](https://lynxjs.org/guide/use-native-modules.html) 和[自定义元素](https://lynxjs.org/guide/custom-native-component.html)来访问。

## 理解双线程模型

如果你使用过 Web Workers，Lynx 的架构会让你感到熟悉。Vue 运行在**后台线程**上，而原生渲染发生在**主线程**上 — 就像将工作卸载到 Web Worker 以保持 UI 线程的响应性一样。不同之处在于，在 Lynx 中这是_默认行为_ — Vue 的协调过程永远不会阻塞主线程。（参阅 [合理使用主线程以保证交互性](https://lynxjs.org/next/blog/lynx-unlock-native-for-more#use-the-main-thread-responsibly-for-interactivity) 了解设计理念。）

在底层，Vue Lynx 使用[自定义渲染器](https://vuejs.org/api/custom-renderer)在后台线程上构建 **ShadowElement** 树。变更会被批量收集为 **ops** 并刷新到主线程。用户交互以**事件**的形式回传。这构成了一个类似于浏览器事件循环的周期：

```
        ┌──────────────────────────────────────────────────────┐
        │                    后台线程                            │
        │  Vue 3 运行时 · 响应式 · 生命周期 · 你的代码             │
        └──────────────┬──────────────────────▲────────────────┘
                  ops  │                      │  事件
                       ▼                      │
        ┌──────────────────────────────────────┴───────────────┐
        │                     主线程                            │
        │  原生元素 · 布局 · 渲染 · MTS 处理器                     │
        └──────────────────────────────────────────────────────┘
```

一个完整的周期 — ops 下发、事件回传 — 就是一个 **tick**。Vue Lynx 的 `nextTick` 会在一个完整周期后 resolve，因此当你的回调触发时，原生元素已经完全创建完毕。这与 Vue 在 Web 端的 `nextTick` 心智模型相同（等待 DOM 更新），只是扩展到了跨线程的场景。

### Vue API：完全兼容

Vue Lynx 直接复用 `@vue/runtime-core`。响应式、组件模型、生命周期钩子、模板指令 — 一切都与 [Vue 3 文档](https://vuejs.org/guide/introduction.html)中描述的一致。请参阅 [Vue 兼容性](/zh/guide/vue-compatibility) 了解逐项特性对比。

### 双线程环境的注意事项

由于你的代码和原生元素运行在不同的线程上，有一些行为需要注意。

#### 生命周期运行在后台线程 — 使用 `nextTick` 等待主线程

与 Web 端一样，`onMounted` 意味着"组件树已构建完成"。不同之处在于，原生元素是在主线程上异步创建的。如果你需要在挂载后与原生元素交互，请等待一个 tick：

```vue
<script setup lang="ts">
import { onMounted, nextTick } from 'vue-lynx'

onMounted(() => {
  nextTick(() => {
    // 原生元素现已就绪
    lynx.createSelectorQuery().select('.my-list').invoke(...)
  })
})
</script>
```

如果你的 `onMounted` 只是设置响应式状态、定时器或数据请求，则不需要 `nextTick` — 这些操作不依赖原生元素。

#### 后台线程只有异步访问 — 使用主线程脚本满足同步需求

从后台线程访问原生元素始终是异步的。模板 `ref` 返回的是 `ShadowElement`（后台线程对原生元素的引用），而 `getBoundingClientRect()` 等布局查询需要使用[异步 API](https://lynxjs.org/next/api/lynx-api/nodes-ref.html)：

```vue
<script setup lang="ts">
import { onMounted, nextTick } from 'vue-lynx'

onMounted(() => {
  nextTick(() => {
    lynx.createSelectorQuery()
      .select('.target')
      .invoke({
        method: 'boundingClientRect',
        success: (res) => console.log(res),
      })
      .exec()
  })
})
</script>
```

当你需要同步访问时 — 流畅动画、手势处理、布局测量 — 请使用[主线程脚本](/zh/guide/main-thread-script)。标记为 `'main thread'` 的函数直接在主线程上运行，可以调用[主线程 API](https://lynxjs.org/next/api/lynx-api/main-thread.html)（如 [`getComputedStyleProperty()`](https://lynxjs.org/next/api/lynx-api/main-thread/element-get-computed-style.html)），监听 `main-thread-bindlayoutchange`，以及通过 `useMainThreadRef()` 访问原生元素：

```vue
<script setup lang="ts">
import { ref, useMainThreadRef } from 'vue-lynx'

const divRef = ref(null)               // ShadowElement（后台线程）
const mtRef = useMainThreadRef(null)    // 原生元素（主线程，仅限 MTS）
</script>

<template>
  <view ref="divRef" :main-thread-ref="mtRef" />
</template>
```

请参阅 [Vue 特性兼容性](/zh/guide/vue-compatibility) 了解各特性的注意事项和不支持的功能。

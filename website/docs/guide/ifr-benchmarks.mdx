## 1. Strategy ladder

> **Takeaway: Element Templates are the inflection of render cost.**
> Plain IFR replay ≈ full render JS (~8–11 ms / 1k els under an interpreter);
> ET cuts it 7–15×; a Vapor-style design is another 2–3× toward the PAPI floor.
> That is why ET ships with IFR by default.

Seven strategies, same logical first screens, byte-identical documents.
Scenes ~1000–1400 els: **static-heavy** (99.7% static), **content** (37%),
**list** (12%).

### Warm render (`--jitless` V8 ≈ MT engine proxy)

| variant | static-heavy | content | list |
|---|---|---|---|
| bg-baseline (No IFR) | 11.86 ms | 9.36 ms | 6.87 ms |
| IFR without ET | 10.94 ms | 8.40 ms | 6.57 ms |
| ifr-direct (prototype) | 8.17 ms | 6.36 ms | 4.89 ms |
| ifr-static-tpl (prototype) | 1.11 ms | 5.29 ms | 4.00 ms |
| **IFR + ET (default)** | **0.74 ms** | **1.26 ms** | **1.44 ms** |
| ifr-vapor (upper bound) | 0.53 ms | 0.55 ms | 0.46 ms |
| papi-floor | 0.54 ms | 0.39 ms | 0.31 ms |

### Cold first run (`--jitless`)

| variant | static-heavy | content | list |
|---|---|---|---|
| IFR without ET | 18.0 ms | 16.4 ms | 11.7 ms |
| **IFR + ET** | **4.2 ms** | **5.3 ms** | **4.9 ms** |
| ifr-vapor | 1.4 ms | 1.2 ms | 0.8 ms |

### Ops payload across the thread boundary

| variant | static-heavy | content | list |
|---|---|---|---|
| without ET | 77.6 KB | 60.4 KB | 45.1 KB |
| **with ET** | **69 B** | **9.2 KB** | **17.1 KB** |

## 2. All-examples sweep

> Bundle gzip **×2.26**, TTI upper bound **×1.36**; 22/23 examples
> byte-identical across configs. Per-example FCP is **flat** in this
> single-process harness (median ×1.04) — IFR needs a real thread boundary
> to show benefit (see unified matrix / VERIFICATION.md).

The sweep also caught real bugs (`SystemInfo` clobber, numeric-style auto-px,
CSS Modules on MT) — worth running for correctness alone.

## 3. Native engine observations

LynxExplorer (SDK 1.4 / PrimJS): 24/25 examples pass with **zero hydration
mismatches**; No-IFR vs IFR+ET screens visually identical (SSIM ≥ 0.9977).
Single-sample cold starts: first content ~0.3 s earlier (gallery) up to
~1.2 s earlier (7guis) with IFR — directional, not a stable bench.

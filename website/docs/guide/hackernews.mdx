# HackerNews Slopfork

We forked Evan You's classic [Vue HackerNews 2.0](https://github.com/vuejs/vue-hackernews-2.0) ŌĆö the canonical "real-world" Vue example ŌĆö as a validation benchmark for Vue Lynx. Since Vue Lynx is built on Vue 3, we used the [Vue HackerNews 3.0](https://github.com/raukaute/vue-hackernews-3.0) community port as our reference.

**Reuse confidence: High.** Most of the code transferred directly ŌĆö component logic, Vue Router, Composition API, and even `<Transition>`/`<TransitionGroup>` work identically. The only changes needed were platform-specific: HTML elements ŌåÆ Lynx elements, `@click` ŌåÆ `@tap`, and `createWebHistory()` ŌåÆ `createMemoryHistory()`.

## Overview of Changes

The Vue HackerNews 3.0 reference is a Webpack-based SSR app using Vue 3 + Vuex 4 + axios. Here's what we changed when porting to Lynx:

| Aspect | Vue 3 Reference | Vue Lynx |
|--------|-----------------|----------|
| **Template elements** | `<div>`, `<span>`, `<img>`, `<a>` | `<view>`, `<text>`, `<image>`, tap handlers |
| **Event handling** | `@click` | `@tap` |
| **Routing history** | `createWebHistory()` | `createMemoryHistory()` |
| **Scrolling** | Window scroll | `<scroll-view>` component |
| **State management** | Vuex 4 | TanStack Query (Tailwind) / Same patterns (CSS) |
| **HTTP client** | axios | `globalThis.fetch` |
| **Build tool** | Webpack 5 + Express | Rspeedy |
| **SSR** | Full SSR with hydration | None (Lynx doesn't use SSR) |

### Notable adaptations:

- **No `v-html`**: Lynx has no HTML parser, so we use a `stripHtml()` utility to convert HTML comments to plain text
- **`<router-link>`**: Wrapped with Vue Router's `custom` slot API since Lynx has no `<a>` tags
- **SVG spinner**: Replaced with a CSS `border` + `animation` spinner (Lynx doesn't support SVG)

## What Stayed the Same

- **Vue Composition API**: `ref()`, `computed()`, `watch()`, `<script setup>` ŌĆö identical
- **Vue Router API**: `useRoute()`, `useRouter()`, `router.push()`, `<RouterView>` ŌĆö identical
- **`<Transition>` / `<TransitionGroup>`**: CSS transition classes work identically
- **Recursive components**: `<Comment>` self-references for nested threads ŌĆö works identically
- **Template directives**: `v-if`/`v-for`/`v-bind`/`{{ }}` ŌĆö all identical
- **CSS**: SCSS, `@keyframes`, `:class` bindings, media queries ŌĆö all work

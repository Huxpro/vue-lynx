# Vue Feature Coverage

Vue Lynx aims to support the full Vue 3 API surface. The core rendering path — Composition API, SFCs, reactivity, directives — is covered by the [basic example](/guide/quick-start) and tutorials.

This page collects examples for Vue features **beyond the core path**: APIs and patterns that are important for broader Vue ecosystem compatibility but may still be experimental or carry caveats on Lynx.

## Options API

Vue 3 ships the Options API alongside the Composition API for backward compatibility.
By default, Vue Lynx enables it (`optionsApi: true` in the plugin), but you can disable it to reduce bundle size:

```ts title="lynx.config.ts"
pluginVueLynx({
  optionsApi: false, // tree-shakes the Options API runtime (~9 kB)
})
```

## Suspense

Vue's [`<Suspense>`](https://vuejs.org/guide/built-ins/suspense.html) displays fallback content while waiting for async components to resolve. On Lynx, `<Suspense>` works with `defineAsyncComponent` for lazy-loading and code-splitting.

:::warning Caveat
**Async `setup()` is not supported.** Top-level `await` in `<script setup>` requires `withAsyncContext`, which vue-lynx does not export. Use `defineAsyncComponent` or load data in `onMounted` instead.
:::

The example below shows two patterns:
1. **Slow async component** — `defineAsyncComponent` with a simulated 1.5s delay, showing the `#fallback` slot while loading
2. **Lazy-loaded counter** — dynamic `import()` for code-splitting, with Suspense as the loading boundary

# Data Fetching

Lynx provides the [Fetch API](https://lynxjs.org/api/engine/lynx-api/global-api.html#fetch) for making network requests. While you can use `fetch()` directly, we recommend [TanStack Query](https://tanstack.com/query/v5/docs/framework/vue/overview) (Vue Query) for managing server state — it handles caching, background refetching, stale data, and mutations out of the box.

See also: [Lynx — Networking](https://lynxjs.org/next/react/data-fetching.html)

## Using TanStack Query (Vue Query)

### Installation

<PackageManagerTabs command="install @tanstack/vue-query" />

### Setup

Register the Vue Query plugin with your vue-lynx app:

```ts title="src/index.ts"
import { createApp } from 'vue-lynx';
import { VueQueryPlugin } from '@tanstack/vue-query';
import App from './App.vue';

const app = createApp(App);
app.use(VueQueryPlugin);
app.mount();
```

### Fetching Data with `useQuery`

Use [`useQuery`](https://tanstack.com/query/v5/docs/framework/vue/reference/useQuery) to fetch and cache server data. It returns reactive refs for `data`, `isLoading`, `isError`, and `error`:

```vue title="src/App.vue"
<script setup lang="ts">
import { useQuery } from '@tanstack/vue-query'

interface Post {
  userId: number
  id: number
  title: string
  body: string
}

const fetchPosts = async (): Promise<Post[]> => {
  const response = await fetch('https://jsonplaceholder.typicode.com/posts')
  if (!response.ok) throw new Error('Failed to fetch posts')
  const data = await response.json()
  return data.slice(0, 10)
}

const { data: posts, isLoading, isError, error } = useQuery({
  queryKey: ['posts'],
  queryFn: fetchPosts,
})
</script>

<template>
  <view>
    <text v-if="isLoading">Loading...</text>
    <text v-else-if="isError">{{ error?.message }}</text>
    <scroll-view v-else scroll-orientation="vertical">
      <view v-for="post in posts" :key="post.id">
        <text>{{ post.id }}: {{ post.title }}</text>
      </view>
    </scroll-view>
  </view>
</template>
```

### Mutations with Optimistic Updates

Use [`useMutation`](https://tanstack.com/query/v5/docs/framework/vue/reference/useMutation) to modify server data. Optimistic updates let you immediately reflect changes in the UI, rolling back automatically if the request fails:

```vue title="src/App.vue"
<script setup lang="ts">
import { useQuery, useMutation, useQueryClient } from '@tanstack/vue-query'

const queryClient = useQueryClient()

// ...fetchPosts and Post interface from above...

const deletePost = async (postId: number) => {
  const response = await fetch(
    `https://jsonplaceholder.typicode.com/posts/${postId}`,
    { method: 'DELETE' },
  )
  if (!response.ok) throw new Error('Failed to delete post')
  return postId
}

const mutation = useMutation({
  mutationFn: () => deletePost(1),
  onMutate: async () => {
    // Cancel in-flight queries so they don't overwrite our optimistic update
    await queryClient.cancelQueries({ queryKey: ['posts'] })

    // Snapshot current state for rollback
    const previousPosts = queryClient.getQueryData<Post[]>(['posts'])

    // Optimistically remove post with ID 1
    queryClient.setQueryData(['posts'], (old: Post[] | undefined) =>
      old ? old.filter((post) => post.id !== 1) : [],
    )

    return { previousPosts }
  },
  onError: (_err, _variables, context) => {
    // Rollback to previous state on failure
    if (context?.previousPosts) {
      queryClient.setQueryData(['posts'], context.previousPosts)
    }
  },
})
</script>

<template>
  <view @tap="mutation.mutate()">
    <text>Delete Post 1</text>
  </view>
</template>
```

## Using `fetch()` Directly

For simple one-off requests, you can use the Fetch API directly with Vue's reactivity:

```vue title="src/App.vue"
<script setup lang="ts">
import { ref, onMounted } from 'vue'

const posts = ref([])
const loading = ref(true)

onMounted(async () => {
  try {
    const res = await fetch('https://jsonplaceholder.typicode.com/posts')
    posts.value = await res.json()
  } finally {
    loading.value = false
  }
})
</script>
```

For anything beyond simple reads — caching, background refresh, pagination, mutations — use TanStack Query instead.

See [TanStack Query — Vue Overview](https://tanstack.com/query/v5/docs/framework/vue/overview) for additional resources.

# TodoMVC

## The Quest

On **day 1** of the Vue Lynx project, we needed to answer a critical question: can Vue 3's Custom Renderer API actually target Lynx's native rendering engine with minimal adapter code?

We chose [TodoMVC](https://github.com/tastejs/todomvc/tree/gh-pages/examples/vue) as our first validation target — the canonical benchmark for comparing JavaScript frameworks. Same app, same functionality, different implementations. If we couldn't make this work, the entire project would be in jeopardy.

## The Result

The entire business logic transferred directly — Composition API (`ref`, `computed`, `watch`), component patterns, and template directives work identically. The only changes needed were platform-specific: HTML elements → Lynx elements, DOM events → Lynx events, and CSS adaptations for Lynx's layout model.

## Overview of Changes

| Aspect | Original (Web) | Vue Lynx | Delta |
|--------|----------------|----------|-------|
| **Components** | 4 SFCs + 1 entry | 4 SFCs + 1 entry | Same |
| **Vue Router** | Required (3 routes) | Not needed | Replaced with `ref('all')` |
| **Business logic** | Composition API | Composition API | Zero changes |
| **Template syntax** | `v-if`/`v-for`/`v-show`/`{{ }}` | Same | Zero changes |
| **HTML tags** | `div`/`li`/`button`/`label` | `view`/`text`/`input` | Mechanical swap |
| **Events** | `@click`/`@keyup.enter` | `@tap`/`@confirm` | Name swap |
| **Checkbox** | `<input type="checkbox">` | `<view>` + `@tap` | Rewrite (no checkbox in Lynx) |
| **CSS** | `todomvc-app-css` | Custom `todomvc.css` | Adapted (no pseudo-elements) |

### What Stayed the Same

- **Vue Composition API**: `ref()`, `computed()`, `watch()`, `<script setup>` — identical
- **Component patterns**: Props, emits, slots — all work identically
- **Template directives**: `v-if`/`v-for`/`v-bind`/`{{ }}`/`v-show` — all identical
- **Reactivity**: The entire `@vue/reactivity` system works unchanged

## How much of Vue do we reuse?

| Layer | Source | Reuse |
|-------|--------|-------|
| **Reactivity** | `@vue/reactivity` | 100% official |
| **Component system** | `@vue/runtime-core` | 100% official |
| **Template compiler** | `@vue/compiler-dom` + `@rsbuild/plugin-vue` | 100% official |
| **Renderer** | Custom `createRenderer(nodeOps)` | Custom (~200 LOC) |
| **v-show** | Custom (ops-based display:none) | Custom (~15 LOC) |
| **v-model** | Stubbed | Not yet implemented |
| **Cross-thread comm** | ops buffer + callLepusMethod | Custom (~250 LOC) |

**Reuse rate: > 95%.** Total custom code: ~500 lines of runtime + ~300 lines of build plugin.

This gave us the confidence to continue building Vue Lynx.

## What this page is

The three older campaigns answered different questions:

| campaign | question | doc |
|---|---|---|
| IFR styles | first-frame cost & FCP | [IFR Benchmarks](/guide/ifr-benchmarks) |
| VDOM vs Vapor | instrumented BG / e2e | [VDOM vs Vapor](/guide/benchmark-vapor) |
| Vue vs React | black-box click→DOM | [Playground](/guide/benchmark-playground) |

They shared **1k→30k labels** but not a **metric scale**. This report puts
every cell in one schema, grades the published claims, and shows
playground-style tinted tables + scaling charts.

## How to read it

1. **Conclusions** at the top — each card is takeaway → so what → verify numbers.
   Easy to digest; tables below are for checking the claim.
2. **Storm table** — IFR × framework interaction latency (same protocol as the playground).
3. **Storm charts** — select / update / create vs N on one host.
4. **FCP table + charts** — content-probe first-frame ladder, including ReactLynx
   Snapshot+IFR on the same card density (separate instrument from storms!).

Never ratio storm ms against FCP ms. Absolute milliseconds are host-bound;
ratios on one host are the portable claim.

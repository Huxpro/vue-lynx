## Codebase Patterns

- **CSS classes, not inline styles**: Use BEM classes via `createNamespace()` + `bem()`. Inline styles ONLY for dynamic `color` prop.
- **Import .less**: Every component MUST `import './index.less'`. The .less file defines CSS variables AND class selectors.
- **@keyframes in .less**: Animations (spinners, transitions) use CSS @keyframes, NOT JS setTimeout hacks.
- **<Transition> needs :duration**: Always pass explicit `:duration` prop — auto-duration doesn't work in vue-lynx.
- **<scroll-view> for scrolling**: Lynx does NOT support `overflow: scroll`. Use `<scroll-view scroll-orientation="...">`.
- **pointer-events: none on masks**: Absolute-positioned overlay/mask elements MUST have `pointer-events: none` or they block touch events.
- **transform for animations**: Use `transform: translateY/X()` for position animations, NOT `height`/`width`/`paddingTop`.
- **Icon font via @font-face**: Use Vant's actual icon font, NOT emoji/unicode fallbacks.
- **Array handler fix**: vue-lynx merges fallthrough event listeners into arrays. The runtime normalizes these automatically.
- **Style values**: Use string px units (`"20px"` not `20`) to avoid deprecation warnings.
- **Unknown icon fallback**: Icon names not in @font-face map fall back to rendering the name itself as text (backward compat with `icon="★"` etc.)
- **Lynx CSS no-inherit**: CSS properties don't inherit parent→child by default. Apply fontSize/color directly to `<text>` elements, not just the parent `<view>`.
- **Function-type prop defaults**: In `withDefaults(defineProps<{...}>())`, function-type prop defaults (e.g., `formatter`, `filter`) must NOT be wrapped in a factory function. Use the function directly: `formatter: (type, option) => option`. Wrapping in `() => ((type, option) => option)` causes the formatter to return a function instead of the processed value.
- **Utils module**: `src/utils/create.ts` (createNamespace+bem), `src/utils/format.ts` (addUnit, isDef, isNumeric), `src/utils/index.ts` (re-exports).
- **Text query in tests**: Vue renders empty `<text>` elements as v-if/slot anchors. Use `querySelectorAll('text')` + `Array.from(...).find(t => t.textContent === '...')` to find the right text element, NOT `querySelector('text')` which may return an empty anchor.
- **No Vue `<Transition>` in Lynx**: `@vue/runtime-dom`'s Transition uses `el.classList` which doesn't exist on Lynx elements. Use CSS transitions directly (inline transitionProperty/transitionDuration + toggled opacity/transform) instead of `<Transition>` component. Fire opened/closed events via `setTimeout(duration)`.
- **Popup animation pattern**: Center position uses opacity fade, side positions use transform slide. All animation state (opacity, transform, transitionDuration, pointerEvents) in inline styles; all visual state (position, round, close-icon) in BEM CSS classes.
- **shallowReactive for child registration**: When storing computed refs in a reactive array (e.g. `reactive([{ checked: isCheckedRef }])`), `reactive()` unwraps the refs making `.value` undefined. Use `shallowReactive([])` to preserve ref objects.
- **Icon fallthrough in Checkbox**: Don't use the Icon component inside Checkbox for the default check icon. Icon wraps in Badge, so `:style` and `class` fall through to Badge's root, not the `.van-icon` element. Render the icon font char directly via `<view class="van-icon"><text class="van-icon__font">...</text></view>`.

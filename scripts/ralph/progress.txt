## Codebase Patterns

- **Test runner**: `pnpm test -- --reporter=verbose Icon` (vitest, no `--testPathPattern`)
- **Test pattern**: Use `defineComponent` + `h()` render, then `container.querySelectorAll` for DOM assertions
- **addUnit helper**: Vant uses `addUnit(size)` to convert numbers to `${n}px` — replicate locally in components
- **Props defaults**: Only set defaults that Vant explicitly sets (e.g. `tag: 'i'`, `classPrefix: 'van-icon'`). Leave size/color unset for inherit behavior.
- **Pre-existing TS errors**: `vue-lynx-testing-library` has no type declarations — this is project-wide, not per-component
- **Style values**: Use string values with units (e.g. `"20px"`) not numbers in inline styles — numeric values trigger deprecation warnings and `getAttribute('style')` may not work on `<text>` elements with numeric values
- **Text element queries**: `querySelector('text')` returns first `<text>` including template whitespace nodes — use `querySelectorAll('text')` and filter by style/content
- **CSS custom properties as inline styles**: Lynx strips `--van-*` from inline styles. CSS variables only work through .less files. Test theme var conversion logic via unit tests on utility functions, not by checking `getAttribute('style')`
- **Exports from `<script setup>`**: Vue SFC `<script setup>` cannot contain ES exports. Extract types, constants, and utilities to separate `.ts` files (types.ts, utils.ts)

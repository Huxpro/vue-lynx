a · Web-friendly

The code you'd write anyway.

  • SFCs, templates, CSS. The same .vue file. Class selectors and CSS variables that just work.
  • HTML-shaped tags. <view>, <text>, <image>, <list> — the mental model is one swap away.
  • Renders on the web too. Every example in the deck is running through Lynx for Web right now.
HelloWorld.vue 
<script setup>
import { ref, computed } from 'vue'

const count = ref(0)
const doubled = computed(() => count.value * 2)
</script>

<template>
  <view class="card">
    <text class="title">Hello, Lynx</text>
    <text @tap="count++">
      Tapped {{ count }} · doubled {{ doubled }}
    </text>
  </view>
</template>

<style scoped>
/* same CSS you write today */
.card { padding: 24px; border-radius: 16px; }
.title { font-size: 28px; font-weight: 600; }
</style>

Web-friendly is a feature, not an accident.

SFCs, templates, scoped CSS — all the same. The element vocabulary is HTML-shaped: <view> for <div>, <text> for <p>/<span>, <image> for <img>, plus <list> when you need virtualization.

Important: the same bundle renders on the web via Lynx for Web — that's how every demo in this deck is running right now.

b · Architecturally performing

Two threads, by design.

Lynx splits your app across a dedicated background thread for framework work and a main thread that owns the UI. Vue Lynx puts the whole reactivity system on one, and lets the other stay free for layout, scrolling, and gesture response.

Background thread

Vue 3 runtime & your code

  • reactivity, lifecycle
  • component tree diff
  • event handlers
  • data fetching
  • flat ops →
Main (UI) thread

Native elements & render

  • ← ops interpreter
  • layout & paint
  • gestures & scrolling
  • Main-Thread Script handlers
  • events →

One bundle, two layers, no JS bridge round-trip on the hot path.

The architectural punchline.

Two threads, both fully programmable in your bundle. Vue runtime — reactivity, diff, lifecycle, handlers — lives on the background. Native layout, paint, gestures, scroll live on the main thread.

They talk via a flat ops buffer, batched per Vue tick. No JS bridge round-trip on the hot path. That's why gestures stay 60fps even while reactivity does its job.

For people from RN-land: this is the part that's different. RN's bridge sits between you and the UI. Lynx doesn't have a bridge — both threads are your bundle.

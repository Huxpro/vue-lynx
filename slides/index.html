The field, in one row. The industry is not short on cross-platform answers: Flutter, React Native, NativeScript, and the Web itself — with its own shell family orbiting it (Electron, Tauri, Ionic). And the newcomer at the end of the row: Lynx.

Vue enters — at the top, looking down the row. Every stack below has a home framework story. Vue — one of the largest frontend communities on earth — has never had a dominant one. Weex tried, NativeScript-Vue tried; nothing stuck.

So let's speed-date the row, one candidate at a time.

self-renderedDart — wrong language

Candidate 1: Flutter. Renders everything itself — Skia/Impeller — and the platform spread is impressive: iOS, Android, desktop, even web. But the only door on top is Dart. Wrong language, wrong ecosystem: Vue can't plug in at all. The wire never connects. (Note the dashed web edge — Flutter's web target is notoriously rough.)

native UI
out-of-treebound to React — wrong model

Candidate 2: React Native. Native UI primitives — philosophically the closest. But the entire chain is bound to React: Metro, the renderer, the programming model. Vue isn't just unwelcome; the door literally doesn't fit. (Web and desktop are reachable — dashed here — but react-native-web and the desktop ports are out-of-tree, flaky-tier support.)

plugs in — model half-survivesnative UIlayout ≠ Web
no Web — low ROI

Candidate 3: NativeScript. This one Vue can plug into — but only halfway: the programming model and the ecosystem don't fully survive the trip, and the rendering model isn't web-like (no real CSS layout engine). Worse, the target list doesn't even include the Web (desktop exists, but dashed-tier) — so you maintain a second codebase for a smaller reach. Low return on the investment.

not native enough

So the community stays on the Web — today's actual choice. It connects perfectly: browsers everywhere, and the shell family (Electron, Tauri, Ionic — un-dimmed around Chrome) papering over the gaps. One problem no shell fixes: it's still not native enough — the ceiling we saw in every demo chapter.

native UI primitives
custom rendering engine

And then Lynx steps out of the row — dead center, right under Vue. The frontend seam is open by design: a web-standard programming model, real CSS, a framework-agnostic contract. The wire connects — short, vertical, solid.

Below it, the reach: iOS, Android, Web and HarmonyOS through native UI primitives; desktop and beyond through the custom rendering engine. Web DX in, native UX out — and that pairing deserves a proper title…

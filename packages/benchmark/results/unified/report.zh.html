结论

标题是 takeaway；含义怎么用；核对是数字。勿跨环境比毫秒。

Vapor 点状更新 ~12.6×

含义 Select = 点状更新。交互密集列表默认 Vapor；单次点击常贴帧地板。

核对 selectStorm@10k 743→59 ms · vs RL ~17× · BG ~9.8×

ReactLynx 创建更快 ~2.14×

含义 Snapshot 擅长一次铺开大量节点。创建 ≠ 更新。

核对 create@10k RL 689 · VDOM 1473 · Vapor 1442 ms

ReactLynx 首帧最低

含义 同密度探针上 RL 最快。Vue 追首帧 → IFR+ET。

核对 FCP@10k RL 228 · IFR+ET 278 · VDOM 310 ms

别裸开 IFR

含义 “−19%”不是常数。中大树无 ET 会变慢；默认 IFR+ET。

核对 vs off：1k -10% · 10k +22%

ET 也加速更新 ~9%

含义 模板 clone 挂载后仍有用。裸 IFR ≈ off。

核对 selectStorm@10k off 743 · IFR 643 · ET 674 ms

Vapor 稀疏命名 ×1 FCP -10%

含义 A2 稳态稀疏有同机 ×1 收益；×4 未成 hedge。Native ET 仍是大头。

核对 dense 102.2 → sparse 91.5 ms · ×4 +3%

创建→RL · 更新→Vapor · 首帧→IFR+ET

含义 Select=点状，Update=批量。产品选择压成这一句。

核对 见下方表；跨表勿用毫秒互除。

报比率，别报绝对 ms

含义 换机器中位数可差 2×+。只带同机比率。

核对 例：旧机 React selectStorm@10k ≈2544 ms；本机 ≈1018 ms。

Storm 随规模变化（1k → 30k）

线性坐标、零基线——看绝对差距，不是对数压缩后的形状。Select 曲线看点状更新；Update 曲线看批量吞吐。

Select storm（点状）vs 行数

点状更新：30 次连续选中的总墙钟。每次 tick 只改选中态。Vapor 几乎压平；React 陡升；IFR+ET 把 VDOM 往下拉。

Update storm（批量）vs 行数

批量更新：50 轮全表/多行更新的总墙钟。创建另算——这里只看持续批量吞吐。

Create vs 行数

ReactLynx 创建仍领先。IFR+ET 通过模板 clone 略微帮助 VDOM 创建。

首帧 FCP — 架构阶梯

相同卡片密度（约 1k→30k 元素）：Vue 旗标矩阵 + 并行 ReactLynx Snapshot+IFR。这是首帧量纲，不能和上面的 storm 毫秒直接比。下表 CPU ×1。
